The Big Question

"What is true AI Agent Design? Why do I feel like most of today's 'agents' aren't really agentic at all?"

Most of what the internet calls "AI Agents" are just glorified scripts wearing a fancy name. Let's see why...

Your Original Framework

You mentioned three pillars: knowledge, skills, and past experience. Let me critique this:

Knowledge
Facts, concepts, understanding the world
Skills
Abilities to take action, use tools
Past Experience
What worked/failed before
Critique of Your Framework

The issue: These three overlap significantly:

  • Knowledge includes both learned facts AND distilled experience
  • Past experience is really a type of knowledge about what happened
  • Skills can be both declarative ("how it works") and procedural ("how to do it")

Better framing: Think of it as what you know (knowledge), how you know what you know (memory systems), and what you can do (capabilities).

True Agent: "Why server crash?" → Thinks: "Last 3 times = memory leak, let me check" → 30 sec.

A Better Framework: What a Real Agent Needs

Instead of overlapping pillars, here's a clearer mental model:

1. A Reasoning Engine (The Brain)

Not just pattern matching, but:

  • Hybrid reasoning -- symbolic + analogical + causal + deductive
  • Uncertainty handling -- hold probabilistic beliefs, not binary know/don't know
  • Self-reflection -- "Am I going in circles? Should I verify this?"

2. Memory Systems (Not Just Storage)

Four distinct types that work together:

Working
↔
Episodic
↔
Semantic
↔
Procedural

These aren't separate silos -- they constantly exchange information

Example: Learning to Ride a Bike

Working: "Balance, pedals moving, eyes forward"

Episodic: "Fell last time braking hard on turns"

Semantic: "Leaning counterbalances centrifugal force"

Procedural: (Just knows how -- can't explain!)

3. Action System (More Than Tool Calling)

  • Tool mastery -- Understands HOW tools work, not just WHAT they do
  • Failure awareness -- Knows what can go wrong, has workarounds
  • Dynamic planning -- Can improvise, not just follow scripts
  • Learning from outcomes -- Updates itself based on what happened
Why "Three Layers" Can Be Confusing

Some frameworks list: Executive / Memory / Cognitive

Problem: Memory isn't a "layer" on top of cognition -- memory is USED BY cognition. It's like saying "the brain has a layer for files and a layer for thinking."

Better: Think of components that work TOGETHER, not stacked layers:

Input/Perception
→
Reasoning + Memory Access
→
Action/Output
→
Learn

The "Unverified Knowledge" Insight

Your point about "unverified but useful memory" is brilliant and often overlooked!

Current AI
"I don't know" OR confident but wrong
Human / True Agent
"70% confident, will verify"
Humans are full of uncertain beliefs:

"Sky looks weird, might rain" -- not proven, but USEFUL.

"Person seems trustworthy by gut" -- not fact-checked, but ACTIONABLE.

This is Non-monotonic Reasoning -- basically absent in today's agents.

How to Make YOUR Agent More Agentic

What you can do TODAY:

1 Add Episodic Memory

Log what actually happened:

{ "event": "api_call_failed", "failure_reason": "rate_limit_exceeded", "lesson": "check_rate_limits_before_calling" }

Now it REMEMBERS: "Last time... failed." Next time, checks automatically!

2 Add Confidence Scores
{ "belief": "user_prefers_short_answers", "confidence": 0.75, "evidence": ["asked_for_summary_twice"] }

Says "75% confident" instead of binary "I know / I don't"

3 Add Self-Reflection

After every task: "What worked? What didn't? What would I do differently?"

Finishes coding → Thinks "took too long on tests, should write first next time" → REMEMBERS!

4 Build Skills Registry with Failure Modes
"search_api": { "failure_modes": ["outdated_info", "max_10_results"], "workaround": "verify_with_second_source" }

Tries search → suspicious date → "might be wrong per registry" → verifies!

5 Add "I Don't Know" Metacognition

Before answering: "Am I confident? What would change my mind?"

Thinks "40% confident" → tells user uncertainty → offers to research more.

The Bottom Line

The mainstream confuses Agent (has intentionality) with Actor (just executes). Without knowledge architecture and associative memory, an "Agent" is just a script in a trench coat.

The Three Questions Every True Agent Must Answer:

  1. How does this agent understand ITSELF? -- Its capabilities, limits, biases
  2. How does it remember the AMBIGUITY of the world? -- Holding unverified but useful ideas
  3. How does it extract INEXPRESSIBLE experience from failure? -- Learning what can't be put into words
The future isn't smarter automation -- it's digital cognitive entities that truly think, remember, and understand.
